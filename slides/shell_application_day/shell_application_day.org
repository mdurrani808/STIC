*** Add function
:PROPERTIES:
:BEAMER_ENV: exampleblock
:END:

=./todo add <task description>= - Add a new task

Specification states:
- Use a simple text format with checkboxes: =[ ]= for incomplete, =[x]= for complete

Useful hints:
- =$*= will give you all the parameters passed into the script as a string
- =echo= and =>>= is useful here to add a message to the =$TODO_FILE=

#+BEGIN_SRC bash
  $ ./todo.sh add "Buy groceries"
  Added: Buy groceries
#+END_SRC

* Show TODOs

** Show TODOs
:PROPERTIES:
:BEAMER_ENV: block
:END:

When TODO is run with no parameters, it should show all the TODOs

#+BEGIN_SRC bash
$ ./todo.sh
     1  ⬜ Buy groceries
     2  ⬜ Finish homework
     3  ⬜ Call dentist
#+END_SRC

*** Useful tips
:PROPERTIES:
:BEAMER_ENV: exampleblock
:END:

- We will need another function for this!
- =*)      [code] ;;= can be used as a wildcard in a case statement
- =cat -n= will add line numbers to the output
- Sed substitution can replace =[ ]= and =[x]= with the ⬜ and ✅
- Be sure to escape the brackets in your substitution commands using the backslash

* Complete TODO

** Complete TODO
:PROPERTIES:
:BEAMER_ENV: block
:END:

=./todo.sh done <task_number>= - Mark task as complete

#+BEGIN_SRC bash
$ ./todo.sh done 1
Completed task #1
#+END_SRC

*** Useful tips
:PROPERTIES:
:BEAMER_ENV: exampleblock
:END:

- Sed is useful here again, with the =-i= flag
- Addressing in sed is useful here!
- If you are on macOS, you may need to add ~""~ after =-i= and before your sed script

#+BEGIN_SRC bash
sed -i "" "something here" "$RANDOM_FILE"
#+END_SRC
